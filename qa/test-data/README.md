# Initial DBF Test Data Catalog

В спринте 1 в репозитории фиксируется каталог тестовых наборов, которые нужно подготовить для автоматических и ручных проверок.

## Planned datasets

- `valid_cp866_pair/`
  - Два валидных DBF-файла в кодировке `cp866`.
  - Ожидается успешная загрузка и получение списка полей.

- `valid_cp1251_pair/`
  - Два валидных DBF-файла в кодировке `cp1251`.
  - Ожидается успешная загрузка и получение списка полей.

- `different_schema_pair/`
  - Два валидных DBF-файла с частично различающимся набором полей.
  - Ожидается успешное чтение структуры обоих файлов.

- `corrupted_file/`
  - Поврежденный DBF-файл для проверки ошибки чтения.
  - Ожидается ошибка preview extraction.

- `wrong_extension/`
  - Не-DBF файл для проверки upload validation.
  - Ожидается ошибка валидации расширения.

## Planned edge cases for later sprints

- `hidden_symbols/`
  - DBF-файлы с пробелами, табами, CRLF и невидимыми символами в значениях.

- `duplicates_keys/`
  - Файлы с неуникальными ключами.

- `field_order_mismatch/`
  - Файлы с одинаковыми полями, но разной очередностью.

- `large_file_profile/`
  - Большой набор данных для профилирования производительности.

## Notes

- Пока фактические DBF-файлы не добавлены в репозиторий.
- При добавлении каждого набора рядом должен появляться краткий `expected.md` с ожидаемым результатом.
